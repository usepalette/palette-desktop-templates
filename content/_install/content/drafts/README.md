# Content Drafts

Pieces in progress, one markdown file each. The thinking and writing stage
before something ships. Name files `YYYY-MM-DD-<slug>.md`.

## Template

```markdown
---
title: Working title
channel: LinkedIn | Blog | Newsletter | Pitch | Copy
status: Draft
date: YYYY-MM-DD
---

# Working title

**Takeaway:** The one thing the reader should walk away with — one sentence.

**Audience:** Who this is for and what they care about.

---

## Draft

[The piece itself. Lead with the insight or problem. One idea per paragraph.
Concrete examples over abstract claims.]

---

## Notes

- Distribution ideas, repurposing angles, open questions.
- Sources for any stats or quotes (don't ship unsourced numbers).
```

## Conventions

- **One takeaway per piece.** If you can't name it, the piece isn't ready to
  draft yet.
- **Channel drives format.** A LinkedIn post and a blog post are different
  shapes — set the channel up front (see `../playbook.md`).
- **Cut, don't pad.** First draft long is fine; refine it down.
